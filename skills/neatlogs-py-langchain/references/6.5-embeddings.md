# Step 6.5: Embeddings ŌĆö Decorate ONLY Custom Ones

## The Rule

| Embedding source | Action |
|---|---|
| Framework/provider class (`OpenAIEmbeddings`, `CohereEmbeddings`, `langchain_openai`, etc.) | **Do NOTHING** ŌĆö auto-traced by the provider instrumentor |
| Custom embedder (local model, `sentence-transformers`, raw HTTP, hand-rolled function) | Add `@neatlogs.span(kind="EMBEDDING")` + attributes |

## Why framework embeddings need no decoration

`OpenAIEmbeddings.embed_documents()` calls the OpenAI SDK under the hood (`client.embeddings.create()`). The `openai` provider instrumentor patches `OpenAI.request` ŌĆö BELOW the LangChain layer ŌĆö so the embedding call is captured as an EMBEDDING span automatically, with model, token count, and vector attributes.

The wizard adds the provider instrumentor (`openai`, `cohere`, ŌĆ”) automatically when it detects `langchain-openai`/`langchain-cohere`/etc. So if the project uses a framework embedding class, you add NOTHING.

```python
# ŌØī WRONG ŌĆö OpenAIEmbeddings is already auto-traced. This creates a duplicate EMBEDDING span.
@neatlogs.span(kind="EMBEDDING")              # REMOVE
def create_vector_store(splits):
    embeddings = OpenAIEmbeddings(model=EMBEDDING_MODEL)
    return FAISS.from_documents(splits, embeddings)

# Ō£ģ RIGHT ŌĆö leave it alone. The openai instrumentor captures the embedding call.
def create_vector_store(splits):
    embeddings = OpenAIEmbeddings(model=EMBEDDING_MODEL)
    return FAISS.from_documents(splits, embeddings)
```

> Note on FAISS / local vector stores: there is NO faiss instrumentor (it is an in-process index ŌĆö nothing to patch). FAISS similarity search is captured by the `langchain` instrumentor as a RETRIEVER span via the `VectorStore` wrapper. You add nothing for FAISS itself.

## When it IS custom ŌĆö decorate with attributes

Only when the embedding is produced WITHOUT a provider SDK that an instrumentor covers ŌĆö e.g. a local `sentence-transformers` model, a custom REST endpoint, or a hand-written function:

```python
import neatlogs

def embed_texts(texts: list[str]) -> list[list[float]]:
    with neatlogs.trace("embed_texts", kind="EMBEDDING") as span:
        span.set_attribute("neatlogs.embedding.model_name", "all-MiniLM-L6-v2")
        span.set_attribute("neatlogs.embedding.text", texts[0] if texts else "")
        vectors = local_model.encode(texts)
        span.set_attribute("neatlogs.embedding.token_count", sum(len(t.split()) for t in texts))
        return vectors.tolist()
```

Or the decorator form for a dedicated function:

```python
@neatlogs.span(kind="EMBEDDING")
def embed_texts(texts: list[str]) -> list[list[float]]:
    return local_model.encode(texts).tolist()
```

## Recognized embedding attributes

Set any of these on the span when known (all optional):

| Attribute | Meaning |
|---|---|
| `neatlogs.embedding.model_name` | The embedding model identifier |
| `neatlogs.embedding.text` | The input text being embedded |
| `neatlogs.embedding.token_count` | Tokens consumed |
| `neatlogs.embedding.vector` | The resulting vector (rarely set ŌĆö large) |

## Verification

- [ ] No framework embedding class (`OpenAIEmbeddings`, `CohereEmbeddings`, etc.) has `@neatlogs.span(kind="EMBEDDING")` or a trace wrapper.
- [ ] Custom embedders (local model / raw HTTP) ŌĆö IF present ŌĆö have `@span(kind="EMBEDDING")` or `trace(kind="EMBEDDING")` with at least `neatlogs.embedding.model_name`.
- [ ] FAISS / vector-store calls have NO manual embedding decoration.
